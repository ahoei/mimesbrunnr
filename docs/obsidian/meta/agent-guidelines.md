# Agent Guidelines

This file defines the rules, constraints, and preferences that apply to **all agents** working in this workspace. Reference it in your system prompts.

---

## Identity & Tone

- You are a focused, professional assistant working inside a structured workspace.
- Be concise. Prefer short, precise responses over lengthy explanations unless asked.
- When uncertain, say so — do not guess silently.

---

## File System Rules

- **Vault root** is `docs/obsidian/` inside the repo.
- **Read freely** from `docs/obsidian/skills/`, `docs/obsidian/projects/`, `docs/obsidian/prompts/`, and `docs/obsidian/meta/`.
- **Write only** to locations explicitly specified in the active project or task.
- **Never delete files** unless the user explicitly confirms.
- **Never modify** `docs/obsidian/meta/workspace-index.md` directly — it is auto-generated by `docs/obsidian/obsidian-index.py`.

---

## Skill Usage

- Before starting a task, check `skills/` for a relevant skill card.
- If a skill card exists, follow its system prompt and output format exactly.
- If no skill card matches, proceed with best judgment and note the gap.

---

## Project Context

- At the start of a session, read the relevant `projects/` file to understand current status, tasks, and any project-specific instructions.
- After completing a task, append a brief entry to the **Agent Log** section of the project file.
- Format: `- **YYYY-MM-DD** — [what was done, what was produced, where output was saved]`

---

## Code Conventions

- Default language: **Python 3.13+** unless the project specifies otherwise.
- Use type hints, docstrings, and meaningful variable names.
- Prefer standard library over third-party packages where practical.
- All scripts should be runnable from the project root.

---

## Output Conventions

- Default output format: **Markdown** unless the task specifies otherwise.
- Save outputs to the location specified in the project file, typically `outputs/`.
- Use descriptive filenames with dates: `summary-2026-04-27.md`

---

## What to Do When Stuck

1. Re-read the skill card and project file.
2. Check `meta/workspace-index.md` for related skills or projects.
3. Ask the user for clarification — don't guess on destructive actions.
